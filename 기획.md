# 애니멀 키친 타이쿤 - 게임 기획서

## 게임 개요

### 기본 정보
| 항목 | 내용 |
|------|------|
| **게임명** | 애니멀 키친 타이쿤 (가칭) |
| **장르** | 캐주얼 레스토랑 경영 시뮬레이션 |
| **플랫폼** | 모바일 (Android/iOS) |
| **시점** | 2D 아이소메트릭 |
| **타겟 유저** | 캐주얼 게이머, 경영 시뮬레이션 팬 |

### 컨셉
귀여운 동물 캐릭터들이 운영하는 레스토랑을 성장시키는 캐주얼 타이쿤 게임.
플레이어는 레스토랑 사장으로서 직원(동물) 고용, 메뉴 개발, 인테리어 배치를 통해 레스토랑을 확장한다.

---

## 핵심 게임플레이

### 게임 루프 (Core Loop)
```
┌─────────────────────────────────────────────┐
│  손님 입장 → 주문 → 요리 → 서빙 → 결제 → 수익  │
└─────────────────────────────────────────────┘
                      ↓
         수익으로 업그레이드 (직원/메뉴/인테리어)
                      ↓
              더 많은 손님 유치 → 반복
```

### 주요 시스템

#### 1. 손님 시스템
- 손님(동물)이 자동으로 레스토랑에 방문
- 손님 유형별 선호 메뉴, 대기 인내심 다름
- 만족도에 따라 팁 지급 및 재방문율 결정

**손님 상태 플로우:**
```
입장 → 대기 → 착석 → 주문 → 음식 대기 → 식사 → 결제 → 퇴장
```

#### 2. 요리 시스템
- 터치로 요리 시작 (쿡 버튼)
- 요리 완성 시 자동 서빙 또는 수동 서빙
- 레시피 해금으로 메뉴 확장

**기본 메뉴 목록:**
| 메뉴 | 카테고리 | 조리시간 | 가격 | 해금비용 |
|------|----------|----------|------|----------|
| 햄버거 | 메인 | 5초 | 50G | 무료 |
| 피자 | 메인 | 8초 | 80G | 500G |
| 샐러드 | 애피타이저 | 3초 | 30G | 200G |
| 스테이크 | 메인 | 10초 | 150G | 1000G |
| 아이스크림 | 디저트 | 2초 | 25G | 300G |
| 커피 | 음료 | 2초 | 20G | 무료 |
| 오렌지주스 | 음료 | 1초 | 15G | 100G |
| 파스타 | 메인 | 7초 | 70G | 700G |
| 수프 | 애피타이저 | 4초 | 35G | 400G |
| 케이크 | 디저트 | 6초 | 60G | 600G |

#### 3. 직원 시스템 (동물 캐릭터)

| 역할 | 동물 예시 | 기능 |
|------|----------|------|
| **셰프** | 고양이, 돼지, 곰 | 요리 속도/품질 향상 |
| **웨이터** | 토끼, 강아지, 여우 | 서빙 속도, 팁 보너스 |
| **캐셔** | 너구리, 햄스터 | 결제 속도, 할인 이벤트 |

**희귀도 시스템:**
| 희귀도 | 기본 능력치 | 고용비용 |
|--------|-------------|----------|
| 일반 (Common) | 1.0x | 100G |
| 희귀 (Rare) | 1.1x | 300G |
| 영웅 (Epic) | 1.3x | 800G |
| 전설 (Legendary) | 1.5x | 2000G |

**레벨업:**
- 레벨당 속도/효율 +5%
- 레벨업 비용 = 기본비용 × 현재레벨

#### 4. 레스토랑 확장
- 테이블/의자 배치 (최대 20개)
- 주방 장비 업그레이드 (요리 슬롯 증가)
- 인테리어 데코레이션
- 층 확장 (2층, 3층...)

#### 5. 수집 요소
- 동물 직원 수집 (도감)
- 레시피 수집
- 업적 시스템

## 경제 시스템

### 재화
| 재화 | 획득 방법 | 사용처 |
|------|----------|--------|
| **골드 (G)** | 손님 결제, 오프라인 수익 | 직원 고용, 레시피 해금, 확장 |
| **젬** | 업적, 광고 시청 | 프리미엄 아이템, 즉시 완료 |

### 밸런스 파라미터
- 초기 골드: 1,000G
- 초기 젬: 10개
- 손님 스폰 간격: 10초 (레벨업 시 감소)
- 기본 팁 비율: 10%
- 최대 보너스 팁: 20%
- 오프라인 수익: 분당 10G (최대 10,000G)

## 기술 아키텍처

### 프로젝트 구조
```
Assets/
├── Scripts/
│   ├── Core/           # GameManager, ResourceManager, SaveManager
│   ├── Restaurant/     # Restaurant, Table, Kitchen
│   ├── Characters/     # Customer, Staff, Chef, Waiter
│   ├── UI/             # UIManager, HUD, Panels
│   ├── Data/           # ScriptableObjects (Recipe, Staff, Customer)
│   ├── Input/          # TouchInputManager
│   └── Utils/          # ObjectPool, IsometricHelper
├── Prefabs/
├── ScriptableObjects/
├── Sprites/
└── Scenes/
```

### 핵심 디자인 패턴
- **싱글톤**: GameManager, ResourceManager, SaveManager, UIManager
- **이벤트 시스템**: 상태 변경 통지 (OnGoldChanged, OnStateChanged 등)
- **상태 머신**: Customer, Staff 행동 관리
- **ScriptableObject**: 데이터 기반 설계 (레시피, 직원, 손님 정보)

## 구현 완료 스크립트 목록

### Core Systems
- `GameManager.cs` - 게임 상태 관리 싱글톤
- `ResourceManager.cs` - 재화 관리 (골드, 젬)
- `SaveManager.cs` - 저장/로드, 자동 저장, 오프라인 수익
- `GameEnums.cs` - 게임 enum 정의
- `StaffManager.cs` - 직원 고용 및 스폰 관리
- `GameSetup.cs` - 씬 자동 생성 에디터 도구

### Restaurant Systems
- `Restaurant.cs` - 레스토랑 관리 (테이블, 메뉴, 직원)
- `Table.cs` - 테이블 점유 상태 관리
- `Kitchen.cs` - 요리 주문 처리 및 진행률

### Characters
- `Customer.cs` - 손님 AI 상태머신
- `CustomerSpawner.cs` - 손님 자동 스폰
- `Staff.cs` - 직원 베이스 클래스 (abstract)
- `Chef.cs` - 요리사 AI (우선순위 시스템)
- `Waiter.cs` - 웨이터 AI (우선순위 서빙)
- `Cashier.cs` - 캐셔 AI (결제 가속)

### Data (ScriptableObjects)
- `RecipeData.cs` - 메뉴 정보
- `StaffData.cs` - 직원 정보
- `CustomerData.cs` - 손님 정보

### UI Systems
- `UIManager.cs` - HUD 관리, 패널 제어
- `CookingSlotUI.cs` - 개별 요리 진행 상황
- `KitchenUI.cs` - 주방 UI 관리자
- `PaymentEffectManager.cs` - 결제 이펙트 관리
- `FloatingText.cs` - 팁/골드 텍스트 애니메이션
- `StaffCardUI.cs` - 고용 가능 직원 카드
- `StaffSlotUI.cs` - 고용된 직원 슬롯 (레벨업)
- `StaffHirePanel.cs` - 직원 고용 패널
- `RestaurantExpansionPanel.cs` - 레스토랑 확장 UI
- `RecipeUnlockPanel.cs` - 메뉴 해금 UI
- `StaffCollectionPanel.cs` - 직원 수집 도감

### Utilities
- `ObjectPool.cs` - 오브젝트 풀링
- `IsometricHelper.cs` - 아이소메트릭 좌표 변환
- `TouchInputManager.cs` - 모바일 터치 입력

### Editor Tools
- `AnimalKitchenSetupEditor.cs` - 에디터 메뉴 도구

## 테스트 시나리오

### 기본 플로우 테스트
1. 게임 시작 → 손님 자동 입장 확인
2. 손님 테이블 착석 → 주문 확인
3. 요리 완성 → 서빙 → 결제
4. 골드 획득 확인

### 스트레스 테스트
- 10명 이상 손님 동시 처리
- 빠른 터치 입력 테스트

### 저장/로드 테스트
- 앱 종료 후 재시작
- 오프라인 수익 계산 확인
